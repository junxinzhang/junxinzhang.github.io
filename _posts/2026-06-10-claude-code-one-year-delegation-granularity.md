今天早上一打开电脑，就收到一条推送：**claude-fable-5 上线了。**

我顺手问了它一句「你和 Opus 4.8 的区别是什么」。它答得很客气——自己是新的「Mythos 级」，定位在 Opus 之上，是现在公开可用最聪明的 Claude，推理、写作、编程都更强。

放一年前，我大概会立刻去翻它的跑分，看它在各个榜单上又涨了几个点。但今天我没急着看——因为我想起来，**今天正好是我用 Claude Code 满一年。**

干脆借这个由头，把我和 Claude 这两年好好捋一遍。捋完发现，这两年最实在的变化，其实不是模型考了多少分，而是一件很朴素的事：**我敢交给它的活，从一个函数，慢慢变成了一整件事。**

## 一、2024 年 3 月：第一次用 Claude，它还只会「写片段」

我最早接触并真正用上 Claude，是 **2024 年 3 月**，那时还是 Claude 3。

现在回头看，那个阶段的 AI 写代码，本质就是「写片段」。我把一段上下文贴进对话框，让它帮我补一个函数、改一个报错、解释一段看不懂的代码。它答得不错，但我交给它的活很小——**小到我必须全程盯着，它写一句我看一句。说白了，它是个聪明的搜索引擎，加一个补全器。**

那会儿你不敢真把活交给它。让它写完一段，你还得自己一行行读、自己复制粘贴、自己跑、出了错再自己贴回去问。真正干活的人始终是你，AI 只是在你卡住的那一下，帮你想一想。

但即便这样，我也已经离不开它了。因为它把「卡住」这件事的成本，降了一大截。

## 二、2024 年 6 月：Claude 3.5 发布，AI Coding 第一次「能用」了

转折发生在 **2024 年 6 月，Claude 3.5 发布。**

这是 AI Coding 第一次真正进入「能用」的区间：它能稳定地写出可以直接跑起来的、成块的代码，而不只是零碎的片段。我交给它的活，第一次从「一个函数」放大到了「一个文件」。

也正是从这时候开始，Cursor 这类「AI 写代码」的产品迅速起飞。整个行业第一次相信——AI 写代码不是拿来演示的 demo，是能真正提效的生产力。

我自己的体感也很明显：从前是「我写，卡住了问它」，3.5 之后变成「它先写，我来改」。起点换了人。这一步看着不大，但它把一个习惯彻底掰了过来——**先让 AI 跑，再在它的产出上做判断，而不是等到自己实在写不动了才去问一句。**

## 三、2025 年中：Claude Code 出现，方向盘第一次交了出去

如果说 3.5 是「能用」，那 **2025 年中我开始用上 Claude Code**，才是这两年真正的转折点。

很多人把 Claude Code 理解成「一个更强的写代码 AI」，这是最大的误会。它改的不是模型有多聪明，而是**模型的形态**。

打个比方。**模型升级，是把副驾驶的水平拉高**——副驾驶再厉害，方向盘还在你手里，你得一直盯着路、随时纠正它。Claude Code 做的事完全不同：它把模型从一个「你贴代码进去、它吐答案出来」的对话框，变成了一个能**自己读文件、自己跑命令、自己改代码、自己看报错再改回去**的 Agent。

差别有多大？我用 AI 写代码的那个老循环是这样的：我描述 → 它给一段 → 我复制粘贴 → 我运行 → 报错 → 我再把报错贴回去。**这个循环里，跑腿的人始终是我，AI 只负责动脑那一下。**

Claude Code 第一次把整个循环交给了它自己：我说一句目标，它自己去读项目结构、自己改好几个文件、自己跑测试、自己看报错、自己再修——干完了回来告诉我结果。**跑腿和动脑，第一次都归了它。我只在起点给个目标，在终点做验收。**

这就是为什么我说，模型升级是量变，Claude Code 才是质变：**它不是让副驾驶更强，它是把方向盘整个交了出去。** 而方向盘一旦能交出去，我敢交给它的活，才有可能从「一个函数」放大到「一整件事」。

再往后，整个 2025 年，是各种 Agent 产品密集涌现的一年，像 Manus 这类「你给个目标、它自己跑完一长串步骤」的东西集中冒了出来。它们背后是同一个信号：**AI 干活的方式，正在从「帮你补全」变成「替你跑完」。**

## 四、这一年，我干活的方式被改了什么

方向盘交出去这一年，我干活的方式被实实在在地改了三处。

**第一，我从「写代码的人」，变成了「定义问题和验收结果的人」。** 过去我大量时间花在敲实现细节上；现在我大量时间花在两头——把要做的事说清楚，以及把它交回来的东西验收好。中间那段最累、最耗手的实现，被它接走了。

**第二，我敢同时开好几条线了。** 既然一件事可以整个交出去、自己不用全程盯着，我就能让它跑着 A，自己去想 B、顺手验收 C。一个人的精力，第一次被「并行」撑开了——这在「写片段」的年代根本不可能，那时每一行都得我盯着。

**第三，也是最意外的：我对它的要求，反而变高了。** 交出去的活越大，对「把问题讲清楚」的要求就越高。一句话说得含糊，换来的就是跑偏一整件事、推倒重来。所以这一年我练得最多的，不是写代码，而是**把一件事拆解清楚、讲明白的能力**——而这，恰恰是 AI 替不了、也最值钱的那部分。

你看，AI 并没有让人变闲。它把人从「手」里解放出来，逼着你把劲全使到「脑」上。**它接走的是体力活，留给你的是判断。**

## 五、所以今天这条 Fable 5 推送，我怎么看

绕了一圈，回到今天早上那条把我拉回现实的推送。

Fable 5 是新的「Mythos 级」，定位在 Opus 之上，模型字符串叫 `claude-fable-5`，上一代旗舰 Opus 4.8 是 `claude-opus-4-8`，推理、写作、编程都更强。换两年前的我，一定会立刻去扒它每一项跑分高了几个点。

但今天我只问一句：**它能不能帮我，把更大的一摊活接走？**

这才是每天拿它干活的人真正关心的事。模型公司去卷榜单，那是它们的竞赛；和我切身相关的，是这一代能不能让我把原本还得盯着的某件事，彻底放心地甩出去。跑分高几个点，我大概率没什么感觉；但「敢交出去的活又大了一圈」，我这一天的干法就被实实在在地改写了。

下面这张，就是今天早上那条推送，和我跟它的第一句对话。两年前我会盯着它最后那行 benchmark 链接看半天，今天我只关心一件事——它能不能让我把方向盘，交得再彻底一点。

## 六、回到普通人：把「敢交出去」这件事，做成开箱即用的产品

捋完这两年，我最大的感受是：**最大的红利，不在于模型又聪明了多少，而在于「敢把活交出去」这件事，正在从程序员的特权，变成每个人都能用的能力。**

Claude Code 让我这样的人敢把一整件开发的活甩出去；而我每天在做的，就是把这种「敢交出去」的踏实感，搬给不会写代码的普通人。

我一个人打造的 [Zaokit](https://zaokit.app)，做的就是这件事——你不用懂任何技术，只要把要做的事说清楚，它帮你高效完成图文创作和 PPT 生成，唯一网站：[https://zaokit.app](https://zaokit.app)。本质上，它把「图文和 PPT」这件原本要你亲手抠半天的活，变成了你可以放心交出去、只管验收的一件事。这正是我这一年在 Claude Code 上体会到的那种解放，换了个普通人也能用的样子。

**Zaokit 的 MVP 版本，本身就是用 Claude Code 做出来的。** 我把「想做一个什么样的东西」讲清楚，剩下的搭框架、写实现、跑起来，整件事都交给了它。换句话说，这是一次完整的闭环：我用「敢交出去」的方式，做出了一个帮普通人「敢交出去」的产品。

除此之外，[cx.zaokit.com](https://cx.zaokit.com)、[cc.zaokit.com](https://cc.zaokit.com)、[gift.junxinzhang.com](https://gift.junxinzhang.com) 以及[更多项目](https://junxinzhang.com/projects.html)，已经融入不少企业的真实工作流，是稳定靠谱、开箱即用的 AI 全家桶——它们做的是同一件事：**让你敢把越来越大的活，放心地交出去。**

## 写在最后

用 Claude Code 满一年，从 Claude 3 算起两年多，我没攒下什么惊天动地的故事，但有一条线一直清清楚楚：**我敢交给 AI 的活，一年比一年大。** 从一个函数，到一个文件，到一整件能交付的事——这条线，比任何一张跑分表都更能说明这两年到底发生了什么。

所以下次再看到「新模型发布」的推送，先别急着去翻它考了多少分。问自己一句就够了：**这一代，能让我把方向盘，交得再彻底一点吗？**
