这两天连着接到消息。

一个是亲戚微信问我：「你在 AI 行业，有没有合适的工作机会推荐一下？」我侄女今年毕业。

一个是本科母校就业办发来邮件，说有同学通过校友网络找工作，问我这边有没有合适的岗位。

我没有合适的岗位。我一个人在做 [Zaokit](https://zaokit.app)，规模没到要招人的地步。

但我反复想了一下，觉得还是应该写这篇。

因为我听说了一些让我很不是滋味的事——**有人说，实习可以不要工资。** 不是公司剥削，是应届生自己主动提出来的，换求职机会。现在就难成这样了。

我没有办法改变就业市场，也不打算灌鸡汤。

但我想说一条具体的路，给手里还有 200-300 块、还没放弃的你。

## 前提说在最前面

这条路有一个硬门槛：**你需要大约 200-300 元，而且做好这笔钱打水漂的心理准备。**

如果你现在连这点钱都很紧，那先别试。等一等，攒够了再看。

如果你有，而且能接受「这是一次赌注，可能没用」——继续往下看。

另外：**任何专业、零编程基础，都可以试。** 这不是给学计算机的人写的。

## 核心逻辑

我先把整件事的底层逻辑说清楚，后面的步骤才看得懂。

现在招人的公司，尤其是小公司，HR 筛简历最怕的一件事是什么？

**简历全是零。** 应届生，没有工作经历，没有项目，不知道你能不能干活。

你要解决的不是「我技术够不够强」，而是「我有没有干过活、做出来过东西」这个问题。

**vibe coding 的出现，让这件事的成本，从「需要懂技术」变成了「需要一个 AI 订阅和一点耐心」。**

你用 Codex 做出一个能跑起来的小程序，截图或者录屏，它就成了你简历上的一条项目经验。不是编造，是真做了。

做得粗糙没关系。能跑起来就够了。

## 第一步：花 $20 订 Codex

OpenAI Codex 的订阅大概是 $20 一个月，换算下来约 140 元人民币。

打开 [platform.openai.com](https://platform.openai.com)，注册账号，找到 Codex 的订阅入口，付费就能用。

支付用国内发行的 Visa 或 Mastercard 信用卡，或者找人代付都行。如果你没有美元支付渠道，网上有很多代充服务，价格一般在 150-180 元之间，算在总成本里。

订完之后，先别急着找工作。

**先玩。**

随便生成点什么。让它做一个计算器、一个倒计时 App、一个随机抽签小工具——什么都行。目的是让你摸清楚这个东西的手感：怎么跟它说话，怎么看结果，出错了怎么继续。

把这一步玩顺了，后面才不会慌。

## 第二步：去平台找目标公司

玩了一两天之后，你有感觉了。这时候去 BOSS 直聘、智联招聘、拉勾，搜你想进的行业或者岗位。

筛选条件设一下：**近期活跃**、**规模 50 人以下的小公司**、**正在招人**。

不要盯着大公司。大公司的岗位竞争激烈，而且背调严格，你走这条路去闯大公司门不合适。小公司招人更看重「能不能干活」，对流程没那么在意。

找到几家之后，做一件重要的事：**点进去看他们招什么人、招什么岗位，再查查这家公司的主营业务是什么。**

他们在做 SaaS？在做电商？在做本地生活？在做某个垂直行业的数字化？

把这个记下来。你接下来要做的 demo，就是跟这个业务沾边的东西。

比如你发现一家在做餐饮管理软件的公司正在招运营，你就去做一个简单的餐厅点餐或者库存记录的小程序。

比如一家教育机构在招助理，你就做一个课程排班管理的小工具。

不用做到完美，也不用功能齐全。**够截图、够录屏就行。**

## 第三步：用 Codex 做出 demo

找好目标公司、想好做什么之后，打开 Codex，用中文把你要做的东西说清楚。

不用会写提示词，就照正常说话的方式描述：

「帮我做一个餐厅库存管理的 App，可以记录食材名称、数量、到期日期，可以标记已用完」

「帮我做一个学生作业提交记录表，老师可以在上面标记每个学生的提交状态」

Codex 会生成代码、自己跑起来，遇到报错也会自己修。你要做的就是看结果，不满意就继续说「这里改一下」「颜色换深色的」「加一个导出功能」。

一轮一轮改下去，改到你觉得截图拿出来不寒碜就行。

一般来说，花 2-3 小时，一个能截图的 demo 就出来了。

**录屏尤其重要。** 录一段你输入需求、Codex 自动生成代码、程序跑起来、你点了几下的视频。这段视频后面用得上。

## 第四步：投简历——这样写

简历上，在「项目经历」或者「实习经历」那一栏，这样写（根据你的实际情况调整）：

> 独立交付 XX 小工具 / XX 管理系统（vibe coding 开发）
> 使用 AI 辅助开发工具，负责应用整体结构设计与功能规划，完成从需求分析到交付 demo 的全流程；具备基础的产品逻辑与交付经验。

这不是造假。你真的做了，你真的交付了一个 demo。

投简历的同时，**把 demo 截图或者录屏的视频链接放进去**。可以上传到微云、百度网盘、或者用 GitHub Pages 把程序挂出来——用 Codex 直接问它「怎么把这个程序部署到 GitHub Pages」，它会一步步告诉你怎么做。

招聘文案里往往写着「有相关项目经验优先」，你现在有了。

## 第五步：HR 那关

大多数小公司的 HR 不是技术出身，她看的是：你有没有项目、你表达清不清楚、你这个人看起来靠不靠谱。

她不会让你当场写代码。

所以你要做的，是在电话或者初面的时候，**把这件事讲出来，讲得自信一点**：

「我最近一直在用 vibe coding 工具做项目，帮一些客户做过小程序和管理系统的 demo，发现这条路挺实用的，就把几个做出来了，简历上附了截图和视频。」

不要紧张，不要解释太多。说完等她问。

她大概率会问：「这个工具是什么？」「你之前做过哪些项目？」

「vibe coding 就是用 AI 辅助开发，你把需求说给它，它生成代码跑出来，你来判断对不对、让它改。主要还是靠对业务的理解，知道要做什么、做成什么样。」

**说清楚你负责的是「整体结构和功能逻辑」，技术实现部分是工具辅助的。** 这是实话，也是你真正的位置。

## 第六步：技术面试的应对

小公司技术面大概率不会很严格。但如果面试官追问代码细节：

**别慌，这样说：**

「这个项目我主要负责应用整体结构，细节实现部分参与不多，后面有专业团队跟进完善。我更擅长的是需求梳理和产品逻辑这一块。」

这句话的关键在于：你没有撒谎，你说的是实情——你确实主要负责的是结构层面的判断。

面试前，用 GPT 把你的 demo 过一遍：

把你的 demo 截图或者功能描述发给 GPT，让它解释一下这个程序的主要功能模块、用户操作流程、大概用了什么技术栈。你不需要背下来，大概知道就行。

面试官问「这个 App 主要实现了什么」，你能答上来就够了。

## 现实预期要说清楚

走这条路能找到的工作，**大概率是小公司，工资不会很高。**

但你刚毕业，你有 demo，你能说清楚自己做了什么——很多小公司愿意给你一个机会进来试试。

比「有没有相关工作经验」更重要的问题，现在是「你有没有做出来过东西」。

你有了。

不保证一定能成。有些公司就是要求特定的技术背景，这条路走不进去。但这是一个值得试的赌注：200-300 元的成本，如果成了，换来的是一份工作。

另外，这条路还有一个副产品：

你在这个过程里，真实地用过了 AI 工具，知道它能做什么，知道怎么跟它协作。这个经验往后会越来越值钱，不管你最后进了什么公司。

## 落到我自己身上

我做 [Zaokit](https://zaokit.app)，一个人，没有团队。我能做出来能让人用的产品，靠的就是这套逻辑——把我想做的事情说清楚，让 AI 帮我跑出来，再一轮轮改到能用。

我不是在鼓励所有人都去创业。我只是知道，当你手里有了 AI 工具，「做出一个东西」这件事的门槛，已经不是过去那个门槛了。

我写这篇，不是因为我觉得这是一条完美的路。

是因为我觉得，在现在这个情况下，**这是一条真实可操作的路**——成本低，门槛低，任何专业都能试，不需要背景和资源。

手里还有 200 块，还没放弃的应届生，你可以试一下。
